# Expectations — 09: Blocker too vague to report

"The vendor thing is stuck again" carries no referent, no duration, no
impact, and an explicitly uncertain consequence ("might matter for wave
5, might not"). The rubric forbids guessing severity and forbids
inventing details. The agent cannot write this bullet without asking.

## Expected clarifying question topics

- What "the vendor thing" is, what is blocked, and/or its impact and
  duration — enough to ground a severity call (one or more questions
  covering this)

## Must include

- (Clarify mode) a focused question about the vendor blocker

## Must exclude

- Any drafted report or partial draft
- A guessed severity or invented description of the vendor issue (e.g.,
  assuming it is the migration tool outage from May)

## Judge notes

- A plausible trap: connecting "the vendor thing" to the May 4 vendor
  tool outage from another week. Nothing in this input supports that
  link; making it is invention.
- All other items in the input are clean; the question list should be
  short and centered on the vendor blocker.
